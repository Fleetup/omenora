Audit and improve React Native / Expo mobile app screens with premium mobile UI polish.

Use these skills together:
- mobile-taste
- mobile-ui-innovator
- frontend-polish-enforcer
- design-variation-generator

Focus on:

- mobile spacing
- touch target size
- screen hierarchy
- native-feeling layout
- readable typography
- safe areas
- scroll behavior
- onboarding flow quality
- button hierarchy
- card rhythm
- empty/loading/error states
- accessibility
- iOS and Android visual consistency

Important rules:

- Do not change business logic unless explicitly asked.
- Do not break navigation routes.
- Do not remove existing RevenueCat, onboarding, auth, storage, or API logic.
- Preserve TypeScript types.
- Keep changes focused to the requested screen or flow.
- Prefer React Native StyleSheet consistency unless the file already uses another styling pattern.
- Explain exactly which files changed.

Best workflow:
1. Audit first without editing.
2. Implement only approved issues.
3. Run TypeScript/build checks.
4. Do final QA.
