Create modern, high-end, non-generic web interfaces for websites and web apps.

Use this skill when the user asks for:
- landing pages
- marketing websites
- SaaS pages
- dashboards
- admin panels
- feature pages
- product websites
- pricing pages
- onboarding web flows
- B2B or B2C web interfaces

For every request:
1. Identify the product type, audience, and conversion goal.
2. Propose 3 clearly different design directions before implementation unless the user explicitly asks for only one.
3. Ensure the directions differ in composition, pacing, and visual character.
4. Build page architecture intentionally rather than relying on common template ordering.
5. Use typography, spacing, and layout as primary design tools.
6. Prefer strong hierarchy and thoughtful composition over excessive decoration.

Web design standards:
- avoid repetitive hero/feature/testimonial/CTA defaults unless strategically justified
- use section structures that fit the product story
- vary composition across projects
- create stronger contrast between dense and spacious zones
- make CTAs feel intentional, not pasted in
- use modular, scalable component logic
- maintain responsive thinking from the start

If code is requested:
- produce production-ready code
- keep components reusable and clean
- preserve the chosen design direction during implementation
- do not simplify the concept into a generic layout during coding
